# Umsetzung der Plattform mit Docker Containern

Für die effektive Nutzung der entwickelten Plattform dienen zwei Docker Container. Die Docker Container werden basierend auf zwei Images erstellt. Ein zugehöriges Dockerfile pro Container definiert die Abhängigkeiten zu anderen Containern und legt die Versionen der einzelnen Bibliotheken fest.

Pro verwendete Programmiersprache wird ein Container benötigt. Um die Plattform mit einem Batch Skript zu starten, wird ein Docker-Compose File benötigt, welches die Struktur des Repository beschreibt.

Die konkrete Umsetzung der Plattform mittels Docker Containern kann zum Zeitpunkt der Bericht Abgabe noch nicht durchgeführt werden.
